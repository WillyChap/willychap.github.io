Variables & Types

Variables & Types

A variable is a name that points to a value. Think of it as a labeled box.

Key Points: - Python decides type automatically - Use descriptive names (temperature_c not t) - Python is case sensitive wind ≠ Wind

Why it matters: Clear naming makes debugging and collaboration easier

Built-in Data Types

A = 10
print(type(A))  # integer

B = 10.0
print(type(B))  # float

C = 10j
print(type(C))  # complex

D = '10'
print(type(D))  # string

E = True
print(type(E))  # boolean

F = None
print(type(F))  # NoneType

Strings

Strings & F-Strings

Strings hold text like names, messages, file paths.

# Creating strings
message = "hello"
name = 'Chapman'

# Combining strings
greeting = message + " " + name
print(greeting)
hello Chapman

F-Strings (Formatted Strings)

F-strings allow for easy string formatting:

temperature = 20.5
location = "Boulder"

# F-string formatting
report = f"Temperature in {location}: {temperature}°C"
print(report)

# With formatting
report2 = f"Temperature in {location}: {temperature:.1f}°C"
print(report2)
Temperature in Boulder: 20.5°C
Temperature in Boulder: 20.5°C

Lists & Indexing

Lists & Indexing Basics

Lists are like expandable arrays. They retain the order you insert items.

Python is 0-indexed:

temps = [15.2, 18.7, 22.1, 19.8, 16.5]

print(f"First temp: {temps[0]}")
print(f"Last temp: {temps[-1]}")
print(f"List length: {len(temps)}")
First temp: 15.2
Last temp: 16.5
List length: 5

Slicing

Slicing lets you grab ranges:

temps = [15.2, 18.7, 22.1, 19.8, 16.5]

print(f"First 3: {temps[0:3]}")
print(f"First 3 (alternate): {temps[:3]}")
print(f"Last 2: {temps[-2:]}")
print(f"Every other: {temps[::2]}")
First 3: [15.2, 18.7, 22.1]
First 3 (alternate): [15.2, 18.7, 22.1]
Last 2: [19.8, 16.5]
Every other: [15.2, 22.1, 16.5]

Dictionaries

Dictionaries: {key: value, …}

Dictionaries store key/value pairs. Perfect for metadata.

station = {
    "name": "Boulder",
    "elevation": 1655,
    "temp_c": 20.5,
    "active": True
}

# Access with brackets (not dot notation!)
print(f"Station: {station['name']}")
print(f"Temperature: {station['temp_c']}°C")
Station: Boulder
Temperature: 20.5°C

Dictionary Methods

station = {"name": "Boulder", "temp_c": 20.5}

# Safe access with .get()
humidity = station.get("humidity", "N/A")
print(f"Humidity: {humidity}")

# Common operations
print(f"Keys: {list(station.keys())}")
print(f"Values: {list(station.values())}")
Humidity: N/A
Keys: ['name', 'temp_c']
Values: ['Boulder', 20.5]

Tip: Missing keys will raise an error. Use .get() for safe access!

Control Flow

Control Flow (if/elif/else)

If statements check if a condition evaluates to True.

temperature = 25

if temperature >= 30:
    print("Hot day!")
elif temperature >= 20:
    print("Pleasant day")
else:
    print("Cool day")
Pleasant day

Comparison operators: ==, !=, <, <=, >, >=

Combine with and, or, not for complex conditions.

Important Python Syntax

Python uses indentation instead of “end” statements:

A = 10
if A >= 10:
    print('A >= 10')
elif 0 < A < 10:
    print('0 < A < 10')
else:
    print('A <= 0')

Rules: - : starts a block - Indent by 4 spaces - Consistent indentation!

Loops & Iteration

For Loops

A for loop iterates over a sequence.

temps = [15.2, 18.7, 22.1]

for temp in temps:
    temp_f = temp * 9/5 + 32
    print(f"{temp}°C = {temp_f:.1f}°F")
15.2°C = 59.4°F
18.7°C = 65.7°F
22.1°C = 71.8°F

Loop Variations

# Loop with range
for i in range(3):
    print(f"Iteration {i}")
Iteration 0
Iteration 1
Iteration 2
# Loop with index
temps = [15.2, 18.7, 22.1]
for i in range(len(temps)):
    print(f"temps[{i}] = {temps[i]}")
temps[0] = 15.2
temps[1] = 18.7
temps[2] = 22.1

While Loops

i = 0
while i < 5:
    print(i)
    i += 1
0
1
2
3
4

Warning: Be careful with while loops - make sure they terminate!

Functions

Writing Simple Functions

A function is a reusable block of code.

def function_name(arguments):
    """
    Docstring: what the function does
    """
    # Function body
    result = # some operation
    return result

USE FUNCTIONS PLEASE - They make code reusable and testable!

Function Example

def celsius_to_fahrenheit(temp_c):
    """Convert Celsius to Fahrenheit"""
    temp_f = temp_c * 9/5 + 32
    return temp_f

# Using the function
print(f"20°C = {celsius_to_fahrenheit(20):.1f}°F")
print(f"0°C = {celsius_to_fahrenheit(0):.1f}°F")
20°C = 68.0°F
0°C = 32.0°F

Remember: Defining a function doesn’t run it - you must call it!

Workflow

Script vs. Notebook Workflow

My Philosophy: Use notebooks to explore, scripts to repeat and share.

Notebook (.ipynb)

✅ Great for exploration ✅ Mix code, text, figures ✅ Good for homework ✅ Teaching tool ✅ Full papers written in notebooks

Script (.py)

✅ Run top to bottom ✅ Repeat & automate ✅ Version control ✅ Command line ✅ High Performance Computing

Typical Workflow

  1. Explore in a notebook until the idea works
  2. Move stable code into functions in a script
  3. Keep the notebook as a clean report that calls your script

Debugging

Debugging Mindset

Bugs are normal. Use them as clues.

  1. Slow down and read the error
    • What line failed?
    • What message do you see?
  2. Compare expected vs. actual
    • Print values, check units and types
    • Check data shapes!
  1. Make the bug smaller
    • Reproduce with tiny example
    • Comment out code until only problem remains
  2. Get help effectively
    • Show the error message
    • Explain what you expected
    • GOOGLE THE ERROR MESSAGE
    • STACK OVERFLOW

Data Structures

Combining Data Structures

Use lists to store many things, dictionaries for details about each.

stations = [
    {"name": "Boulder", "temp_c": 20.5, "obs": [20.1, 20.3, 20.7]},
    {"name": "Denver", "temp_c": 22.3, "obs": [22.0, 22.5, 22.4]}
]

# Access nested data
print(f"First station: {stations[0]['name']}")
print(f"First observation: {stations[0]['obs'][0]}")
First station: Boulder
First observation: 20.1

Looping Through Complex Structures

stations = [
    {"name": "Boulder", "temp_c": 20.5},
    {"name": "Denver", "temp_c": 22.3}
]

for station in stations:
    print(f"{station['name']}: {station['temp_c']}°C")
Boulder: 20.5°C
Denver: 22.3°C

Documentation

Documentation & Comments

There is no excuse not to comment your code.

You are explaining to your future self and teammates.

Comments

# This is a comment
temp_c = 20  # inline comment

# Function with docstring
def convert_temp(temp):
    """
    Convert Celsius to Fahrenheit

    Args:
        temp: Temperature in Celsius
    Returns:
        Temperature in Fahrenheit
    """
    return temp * 9/5 + 32

Best Practices: - Comment the “why” not the “what” - Keep them updated - Use docstrings for functions - Be concise but clear
